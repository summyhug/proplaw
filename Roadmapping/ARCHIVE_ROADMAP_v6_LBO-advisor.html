Tab 01 — North Star

MVP Definition

Core Hypothesis

Germany's building law is fragmented across 16 sovereign state codes (LBO). Architects, planners, foreign investors and Bauherren have no unified, queryable interface to navigate this complexity — especially across language barriers.

Our differentiator: A dual-language (DE/EN) AI advisor combining FAISS vector retrieval and a NetworkX knowledge graph to surface precise, cross-referenced answers from all 16 LBOs — with transparent source attribution and explicit out-of-scope detection.

Target Users — Revised

Who uses this?

Primary

Architects & Planners

Need fast cross-Bundesland comparison of LBO rules. Often work in English in international firms. High query complexity.

Primary

Bauherren & Developers

Initiating construction projects. Need to understand Abstandsflächen, GFZ, building class rules per state before engaging consultants.

Secondary

Foreign Investors

Evaluating real estate development across multiple German states. English-first users. Benefit most from bilingual interface.

Secondary

Municipalities & Officials

Compare own LBO rules to neighbouring states. Look up specific §§ during planning approvals. German-first users.

Out of Scope

General Tenants

Wohnraummietrecht / WEG questions are Phase 2. Out-of-scope detection node redirects these users appropriately.

Tab 03 — Feature Spec

Dual Language — DE / EN

NEW in v6.0 — Phase 1 Must-Have Sprint Week 3
Architecture

Translation Loop (LLM-driven)

User Query (DE or EN)
  ↓ // language detection node
Normalise → Internal Processing in DE
  ↓ // FAISS + KG retrieval (always German corpus)
Retrieve relevant LBO §§ chunks
  ↓ // synthesis node
Draft answer in DE
  ↓ // translation node (only if user_lang = EN)
Translate answer → EN // Groq LLM, legal-aware prompt
  ↓
Render response in user's chosen language
  ↓ // source attribution always shows original DE §§
Source panel: §§ always shown in DE with EN label
Design decision: The corpus stays German-only. We do not translate source documents. Translation happens only at the response layer — this keeps the retrieval clean and avoids legal accuracy risk from translated §§ citations.

R1 LBO amendment divergence
16 states update independently — corpus may go stale mid-project		 HIGH Pin to a specific version_date per Bundesland at ingestion. Store version_date in chunk metadata. Note in UI: "Based on LBO version as of [date]". Open
R2 Cross-§§ retrieval gaps
LBO §§ heavily reference each other — FAISS alone misses relational context		 HIGH Knowledge Graph (F2) is the direct mitigation. This is the core architectural differentiator. KG must be complete before Week 3 eval. Mitigated (F2)

Legal terms may be mistranslated or paraphrased incorrectly in EN mode		 HIGH Translation prompt enforces: preserve §§ references verbatim, no paraphrase of cited text, legal-register language. Eval metric: language parity <10% delta. Fallback: show original DE alongside EN. Open — eval in Wk 3

Some state portals don't offer clean PDF downloads; OCR may be needed		 MED Start acquisition Week 1. Prioritise states with clean HTML/PDF exports (Bayern, NRW, BW). Use Gemini OCR as fallback for scanned PDFs. Open — Week 1 task
