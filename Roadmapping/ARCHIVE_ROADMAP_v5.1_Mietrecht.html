MVP Definition
Coach framework: Goal · Hypothesis · Deliverable summary
1 MVP Goal — User + Learning Focused
✅ Deliverable — One-paragraph MVP Summary

Build a deployed, mobile-optimized AI legal advisor that allows tenants, landlords, and property owners in Germany to ask questions in plain German and receive accurate, cited answers grounded in Wohnraummietrecht (BGB §§535–580a) and WEG condominium law. The agent guides users through a structured intake to understand their specific situation, retrieves the relevant legal paragraphs from a curated corpus, and responds with a plain-language explanation plus the exact § reference — never answering without a source. The MVP validates whether users trust AI-generated legal answers when citations are explicit and the system transparently acknowledges the limits of its scope.

2 Core Hypothesis to Validate

IF users can describe their tenancy or property situation in plain German and receive an answer generated by a combined agent architecture — where KG-enhanced retrieval (NetworkX) expands on FAISS seed results to surface legally connected paragraphs that pure vector search would miss —

THEN they will rate the AI's response as more trustworthy and complete than both a general web search result and a standard RAG-only answer — because explicit § citations make the legal basis visible and verifiable, and the KG layer demonstrably reduces critical omissions in multi-paragraph legal situations.

Validation method: Post-session trust rating (1–5) from the 3 AI-simulated personas run on the Lovable prototype, plus at least 5 real test users in Week 3. Secondary metric: omission rate — RAG-only vs KG-enhanced on 10 complex multi-§ queries.

Knowledge Graph Architecture
The differentiator — NetworkX KG layer alongside FAISS RAG · Owner: Sumit + Sebastian
🧠 Why KG — The Pitch Argument

Standard vector RAG retrieves semantically similar text — but German property law is a network of interdependencies. §573 (Kündigung) only makes sense alongside §573c (Fristen) and §573a (Ausnahmen). A pure RAG system can return §573 and miss §573c entirely — a critical omission in a legal context. A knowledge graph layer explicitly models these relationships, enabling the agent to traverse connected paragraphs and return complete, omission-resistant answers. This is our architectural differentiator versus existing legal chatbots.

1 Architecture — Dual Retrieval Pipeline
Track A — RAG (Safety Net)
FAISS vector index → semantic similarity search → top-k chunks → LLM generates answer → citation output.

Owner: Sebastian
Deadline: Working by end Week 1
Fallback: If KG fails, this ships alone
Track B — KG-Enhanced (Differentiator)
FAISS retrieves seed §§ → NetworkX graph traversal finds related nodes → expanded context sent to LLM → richer, omission-resistant answer.

Owner: Sumit
Deadline: Proof-of-concept by end Week 2
Demo: Side-by-side comparison in pitch
Integration point: LangGraph router node decides which retrieval track to invoke based on query complexity. Simple factual queries → RAG. Queries involving multiple parties, deadlines, or exceptions → KG-enhanced.
2 Corpus Document Schema — Sebastian's Output, Sumit's Input
# Every corpus document Sebastian produces follows this schema
{
  "paragraph": "§573",
  "domain": "Mietrecht",  # Mietrecht | WEG | BetrKV | Mietpreisbremse
  "rule_type": "Recht",  # Pflicht | Recht | Ausnahme | Frist | Definition
  "entities": ["Vermieter", "Mieter"],
  "relationships": [
    {"from": "§573", "type": "requires", "to": "§573c"},
    {"from": "§573", "type": "exception_in", "to": "§573a"},
    {"from": "§573", "type": "related_to", "to": "§574"}
  ],
  "text": "Der Vermieter kann nur kündigen, wenn..."
}
Relationship types
requires — must be read together
exception_in — carves out exception
related_to — contextually linked
superseded_by — newer ruling applies
defined_in — term defined here
Rule types
Pflicht — obligation
Recht — right / entitlement
Ausnahme — exception
Frist — deadline / time limit
Definition — legal definition
Entities
Mieter — tenant
Vermieter — landlord
Eigentümer — property owner
WEG-Gemeinschaft — owners' association
Verwalter — property manager
